Bug surfaced — V8 PHASE 7 FOLLOW-UP #6:
  q=host:ip-172-31-26-27 class:1007 since:24h returns 'search failed'
  Root cause: ParseEventsQuery's 'host:' axis writes the literal value
  into ParsedQuery.HostID. Downstream ch.SearchEvents calls
  uuid.Parse(filter.HostID) which rejects hostnames. The parser
  should either (a) UUID-validate the host: value at parse time,
  or (b) hostname-resolve via pg.GetHostByName before the CH
  query (paralleling the JSON API's host_name handling, #120(d)).
